[1]Ganhando Tração com Alias para Git

Criar aliases no Git pode realmente ajudar a aumentar a produtividade ao tornar os comandos mais curtos e fáceis de lembrar. Podemos criar aliases personalizados para os comandos Git no arquivo .gitconfig.

Acessar o arquivo .gitconfig: Digite o seguinte comando no Terminal para abrir o arquivo .gitconfig no seu editor de texto padrão (geralmente é o Vim ou o Nano): nano ~/.gitconfig.

Insira os alias por exemplo: 

[alias]
  co = checkout
  ci = commit
  st = status
  br = branch
  lg = log --oneline --decorate --all --graph
  aa = add
  pl = pull
  ps = push

Caso queira ter mais facilidade neste processo, e uma das coisas que pode lhe acontecer é esquecer os alias, um pequeno shell para fazer busca dos alias git-allias.sh, pode ser bem útil.
